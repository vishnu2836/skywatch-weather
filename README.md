What is SkyWatch?

SkyWatch is a real-time weather web application that I built to give users a beautiful and accurate way to check the weather for any city in the world. Instead of opening a cluttered weather website filled with ads and unnecessary information, SkyWatch gives you exactly what you need — current temperature, hourly forecast, and a 5-day outlook — all in a clean, animated interface that feels premium and modern. The goal was simple: make checking the weather feel enjoyable, not boring.

Why I Built This
Most weather apps I came across were either too overloaded with features or too plain and dull to use. I wanted to build something that looked great, worked fast, and actually taught me how to connect a real frontend to a proper backend with live API data. SkyWatch became my full-stack learning project where I combined everything — a Node.js/Express server, REST API integration, dynamic DOM manipulation, and a fully animated UI — all from scratch. It also helped me understand how real-world applications handle user data, search history, and API communication.

What I Built and What Changed
I started with a simple HTML page that called the OpenWeatherMap API directly from the frontend. The problem with that approach was that the API key was fully exposed in the browser, which is a serious security risk. So I moved all the API calls to a Node.js/Express backend running on port 5000, which acts as a middleman between the user and OpenWeatherMap. Now the frontend never touches the API key directly — it only talks to my own server endpoints.
I also added a Geocoding layer so that when a user types a city name like "Springfield", the app resolves it to the correct coordinates and returns the proper city name with its state and country — for example, "Springfield, IL, US" — instead of returning a wrong or ambiguous result.
For the search history, I started with a simple in-memory array on the server. I also built a complete SQLite database module (db.js) that is ready to be wired in for persistent history that survives server restarts. The UI stores the last 10 searches and lets users click any past search to instantly reload that city's weather without typing again.
The frontend went through the biggest transformation. I replaced a basic card layout with a full glassmorphism design featuring animated floating clouds built with inline SVGs, flocking birds that fly across the screen, glowing background orbs, and smooth fade-up animations on every card. I also added a Google Weather-style info bar that shows precipitation, humidity, and wind speed at a glance, along with a color-coded hourly temperature bar chart that visually shows whether the coming hours will be cool, warm, or hot.

How It Is Useful for People
SkyWatch is useful for anyone who wants a fast and distraction-free way to check the weather. You can search any city in the world and immediately get the current temperature, what it feels like outside, humidity levels, wind speed, and air pressure. If you are planning your day, the hourly forecast shows you how the temperature will change over the next 8 hours with a visual bar chart so you can decide when to go out. If you are planning a trip or the week ahead, the 5-day forecast gives you a daily summary so you can prepare in advance.
The search history feature is especially handy for people who check the same cities regularly — for example, if you travel between two cities for work, your previous searches are saved and one click brings the weather back instantly. Everything loads quickly, works on mobile, and has no ads or popups getting in the way.

What I Used to Build This
For the backend, I used Node.js as the runtime environment and Express.js to build the REST API server. All HTTP requests to OpenWeatherMap are handled using Axios, and CORS is configured so the frontend can communicate with the backend without any cross-origin errors. I also set up a SQLite database using the sqlite3 package and a dedicated db.js module to handle persistent search history storage. During development, I used Nodemon so the server automatically restarts whenever I make a change.
For the frontend, I used plain HTML5, CSS3, and vanilla JavaScript with no frameworks. The design uses CSS custom properties for theming, backdrop-filter for the glassmorphism blur effect, and keyframe animations for everything from the drifting clouds to the flapping birds. Typography is handled by Google Fonts using Bebas Neue for the logo and DM Sans for all body text. Weather icons are loaded directly from the OpenWeatherMap CDN.

Technologies Used
On the backend side, the server is powered by Node.js, which provides the JavaScript runtime environment. Express.js is used to create the REST API with clean route handling for weather, forecast, hourly, and history endpoints. Axios handles all outgoing HTTP requests from the server to the OpenWeatherMap API, and CORS is configured to allow the frontend to communicate with the backend without any browser security issues. Nodemon is used during development to automatically restart the server on file changes. SQLite3 is used as the database layer through a ready-to-use db.js module that sets up a searches table for storing persistent search history.
On the frontend side, the entire UI is built with HTML5, CSS3, and vanilla JavaScript (ES6+) — no frameworks or libraries. All animations including the drifting clouds, flapping birds, glowing orbs, and card fade-ins are built purely with CSS keyframes and the Web Animations API. Google Fonts provides the typography, using Bebas Neue for the logo and DM Sans for body text. Weather condition icons are served directly from the OpenWeatherMap icon CDN.
For data, the app integrates three OpenWeatherMap APIs — the Current Weather API for live conditions, the 5-Day Forecast API for daily and hourly data, and the Geocoding API to resolve city names into coordinates. All API calls are routed through the Express backend so the API key is never exposed to the browser.

How to Run It Locally
Clone the repository and run npm install to install all dependencies. Open server.js and replace the API key placeholder with your own free OpenWeatherMap API key. Start the backend with npm start (or npm run dev for auto-restart). The server runs at http://localhost:5000. Open index.html using VS Code Live Server and the app will be fully functional. Before pushing to GitHub, move the API key into a .env file and add .env, node_modules/, and weather.db to your .gitignore.
